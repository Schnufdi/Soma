Brutally honest verdict — Revision 5
The defining gap is closed.
Soma is now a genuinely adaptive coaching system.
This revision does one thing that the previous four could not claim: it closes the architectural gap that separated Soma from a truly adaptive product. The weekly reconciliation pass is built and live — a full Claude call reading the last 7 day logs with multi-week history and strength records, outputting structured programme adaptations (intensity scheme, volume delta, day swaps, rest insertions, macro targets), and applying them to the next week's plan with a confirmation step and before/after diff. The data is no longer just captured. It now changes the programme.

Beyond the reconciliation, this revision fixes the day browsing regression (a silent isTraining flag error that made training blocks invisible when browsing non-current days), upgrades four UX scenarios from partial/fail to full pass (travel mode, 30-minute sessions, sleep-adaptive intensity, day browsing), and ships a raft of daily plan quality improvements: contextual status chips, time selector, evening prompt bar, day-close card, meal slot labels, and the sleep banner. The mealbuilder dead-end has been cleaned up. The UX scenario coverage now passes 12 of 24 scenarios outright with 4 partial — up from 8 pass at Revision 4. The product is now broadly deployable and genuinely differentiated. The only remaining pre-launch blocker is the auth gate.

Built — Adaptive Intelligence
Weekly reconciliation v2 — the feedback loop is live
The single most important thing left to build in this product is now built. dp-reconcile.js v2 reads the last 7 day logs with RPE data, energy ratings, sleep flags, and completion status — plus 4 weeks of compressed history from behaviourMemory.weekSummaries and the full strength record from bl_ex_weights. The Claude call returns a JSON schema with typed structured actions: swap_days, add_rest, set_intensity, set_volume, set_protein, set_calories, note_only. Each action carries the data fields needed to execute it. The apply function executes all auto-applicable actions against the existing week override system — writing to next week's key specifically so the current week is never disrupted. Confirmation modal shows a before/after diff. Cost: ~$0.03 per weekly pass. The product is now genuinely adaptive.
Built — Daily Plan UX
Day browsing, travel mode, time selector, sleep banner, contextual chips
Five daily plan quality items shipped together. Day browsing: fixed a silent regression where isTraining = t.isTraining (today's flag) was used for all rendered days — training blocks for browsed days were filtered by the wrong act structure and simply vanished. Fix: derive isTraining from the browsed plan's own trainBlock. Travel mode: one-tap activation restructures the day around hotel gym or bodyweight options with no rest days counted against the week. Time selector: chip row lets user declare 30 / 45 / 60 / 90 min availability — plan condenses or expands accordingly. Sleep intensity banner: when sleep quality is logged as poor, checkSleepIntensityBanner() fires a coach note suppressing high-RPE sets. Contextual chips: status indicators for hydration, supplement timing, and energy state — 2 of 5 fully wired.
Built — Scheduling Intelligence
Salvage week mode + compensation hacks
When a user has missed multiple sessions in a week, Soma now enters salvage week mode — condensing the remaining training into the available days rather than continuing to show a failing week plan as-is. Compensation hacks extend this: if a strength session was missed, the system can shift volume to a later session or convert a rest day to a lower-intensity make-up. This addresses one of the highest-churn UX moments in fitness apps — the user who misses Tuesday and feels the whole week is "ruined". The UX scenario W2 (three missed sessions, 2 days left) now passes fully.
Built — Daily Plan Completeness
Evening prompt bar, day-close card, meal slot labels
Three small-surface items that close real behavioural gaps. Evening prompt bar: appears after 7pm if no session log exists for the day — a soft nudge to complete the evening check-in. Day-close card: end-of-day summary card showing training compliance, protein vs target, and a single coaching line. Surfaces the day's win rather than leaving the UI silent at close of day. Meal slot labels: meals in the daily plan are now labelled by slot (Pre-workout / Post-workout / Lunch / Dinner) rather than generic numbers — reduces cognitive load when scanning the plan for what to prepare next.

03 Critical gaps — current state
What remains
genuinely thin.
The defining architectural gap — the feedback loop — is now closed. One pre-launch blocker remains. Two medium-priority items remain open.
Urgent — pre-launch blocker
Auth gate — all pages still accessible without login
The authentication infrastructure is fully built and working. The gate is not enforced. Any URL in the app is accessible without a Supabase session. This was acceptable for development; it is a blocker before public launch with paying users. One redirect check in each page's init sequence. The pattern exists in auth-protected pages — needs rolling out systematically. Must ship before any paid onboarding begins.
Partially addressed
Science library evidence audit — still unstarted
The evidence tier framework is live on accelerators and supplements. The science library (15+ pages covering sleep, gut health, longevity, hormones, metabolism) predates the framework and hasn't been audited. Individual pages still make claims with more certainty than the research supports. A systematic pass applying the same tier methodology — adding evidence badges and updating coaching instructions — is the remaining scientific rigour work. Medium effort, high trust impact for the sophisticated user segment Soma is positioned for.

07 Differentiation — Revision 5
What makes it
defensible now.
The moat has moved from structural-but-potential to structural-and-operational. The adaptive loop is live. The combination of capabilities Soma now ships is genuinely novel in the consumer fitness space.
New moat — live adaptive intelligence
Soma reads your training history and changes your programme. No competitor does this.
Whoop tells you your recovery score. Levels tells you your glucose. Renaissance Periodisation gives you a static programme template. Future puts a human coach on async messages. None of them read your actual logged data — RPE, sleep, missed sessions, energy ratings, strength progression — and output programme adaptations for next week as a machine-executable action sequence. Soma now does. The weekly reconciliation pass performs temporal reasoning that closes the loop between data capture and plan change. This is now live, not planned.
Existing moat — brand & pricing position
Soma is positioned against PT coaching, not consumer trackers
Soma at £39/month benchmarks against a PT session. One PT session = one month of daily structured intelligence + an adaptive loop that reads 7 weeks of history and adjusts next week's programme. No competitor makes this comparison. The positioning is uncontested and is now backed by a capability that justifies it.
Existing moat — epistemic honesty
No competitor labels claims by evidence quality simultaneously across the full product
Four-tier evidence framework on all accelerators and supplements. Risk classifier pre-screen on 10 health categories. Global glossary explaining scientific terms in-context without leaving the product. The epistemic architecture is more sophisticated than any consumer fitness product. Science library audit is the remaining work here.
Structural advantage
The operating system positioning is now more than positioning
Training, nutrition, sleep, recovery, supplementation, science education, meal prep, adaptive coaching — in one product, with data flowing between surfaces. Day log data feeds reconciliation. Reconciliation feeds next week's plan. Programme adaptations feed session defaults. The circular data flow is now operational, not theoretical. Soma is the only product in the space where using it longer makes it meaningfully better at coaching you.
